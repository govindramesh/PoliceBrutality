Predicting Police Violence in US Counties Using Machine Learning Algorithms

Using machine learning skills to identify and bring awareness to systemic problems of society.

Rationale

Over the past 20 years, there have been almost 27,500 reported deaths as a result of police brutality, an average of over three a day. With the shooting of Michael Brown in 2014 and the Black Lives Matter movement, the topic of police brutality came to the forefront of politics. However, despite attempts to reduce police violence, the number of deaths continues to increase every year. This project is trying to provide insights as to where and why police brutality occurs, and could help reduce the number of incidents each year.

Research Question

The objective of this project is to use a machine learning algorithm to create a model predicting the number of deaths as a result of police violence in U.S. counties. As cities continue to develop and change, the results of this model can help determine the need for further supervision of police-civilian interactions and aid in decisions to allocate resources and training in order to prevent police brutality.

History of Police Violence

Throughout American history, accounts of police violence show up repeatedly. Even with protests and movements fighting against the injustice seen with police violence, the incidents haven’t stopped. In fact, they continue to increase each year. In 2019 alone, there were over 1000 deaths as a result of police interaction, up from 995 known reportings in 2018. Many believe that police violence is a result of underlying racism, and while this project may support or refute this claim, it may not be accurate in terms of reality since it only draws conclusions from demographic, economic, and social data that is publicly available, not the motives of police officers. The model created will not show complications with specific individuals, instead identifying trends on larger scales to provide insight as to where police violence occurs.

Machine Learning Algorithms

Machine learning is a type of artificial intelligence which creates a model that can automatically learn from provided data without being manually programmed. The process of machine learning begins with large amounts of data, which can then be analyzed to look for trends or patterns. Then, the algorithm can make generalizations and conclusions about input variables. There are two types of machine learning—supervised learning and unsupervised learning. For this project, supervised learning was the correct path, since there was already data on what I was trying to determine: the number of deaths as a result of police violence in each U.S. county.

Supervised Learning:

  • Uses data with known outputs to create an inferred function which generalizes the input variables.
  • Using data with known outputs that was not used to train the model (testing set), the model can be tested for accuracy by comparing its predicted output to the actual output.
  • After training, the model can predict the outputs with new data points for which the output is unknown.

Decision Tree:
Uses a series of decisions to narrow down the possible outputs until a final prediction is made. Each node on the tree represents a decision, where the value of one of the input variables for a data point determines the next node it reaches. The last node on each path, or branch, is called a leaf, and gives the prediction value for its branch.

Random Decision Forest:
Merges several decision trees together, averaging their predictions (sometimes weighted based on the predicted error) to create a more accurate and stable final prediction. Random Decision Forests reduce the chances of overfitting, in which a decision tree becomes too specific and “fits” too closely to the training data, not allowing the model to create generalizations.

Gradient Boosted Regression Tree:
Creates an ensemble of weak decision trees in which each successive model targets the weaknesses of the previous models.

Linear Regression:
Creates a function to try and predict a value based on multiple input variables. During training, the algorithm learns the weights for each input variable, which will be the coefficients of the variable in the regression function.

Methods

Materials

Data was collected from online databases such as The Washington Post for reports on police killings across the United States since 2013, the Census Bureau’s American FactFinder for demographic and economic information of counties, and the FBI’s UCR Program for annual crime levels. The raw data was prepared using Google Sheets and Jupyter Notebook, and then used to create and train the machine learning algorithms using BigML software.

Procedure

  • Data collection and preparation
    • Download raw data from databases as CSV (comma-separated values) files
    • Import data into Google Sheets or Jupyter Notebook
    • Convert cities to counties as needed
    • Delete duplicates, erratic data, etc.
    • Finish with values in a specific field for each county
  • Creating machine learning model
    • Import data into BigML
    • Create a dataset
    • Split dataset into training(80%) and testing(20%)
    • Choose an algorithm and configure settings to create each model with the training dataset
    • Repeat with different algorithms (decision tree, gradient boosted regression trees, random decision forests, linear regression), different training settings, and different fields selected
  • Testing and evaluating models
    • Set up the “Evaluate” program in BigML to run the model against the testing dataset
    • Record the mean absolute error, mean squared error, and r2 value

Data Analysis

The "Evaluate" program in BigML returns three measurements of a model's accuracy—r2, mean absolute error, and mean squared error—in comparison to the mean prediction and random prediction. These measurements will help determine the most effective model and how effective it is. I will also be using scatterplots with linear regression lines to compare each variable to the objective field(# of shootings) in order to observe the possible correlations. I am using the Pearson correlation coefficient to analyze the strength of relationships between each data field and the number of shootings per county, which will help me identify trends and draw conclusions in addition to the machine learning model.

Conclusions

After testing various machine learning algorithms, I found the best to be a Random Decision Tree. My model had a mean absolute error of 1.41 and a mean squared error of 9.65, which represent how far off the model was from accurately predicting the number of police shootings in a given county. The r2 value was 0.86, which means that the model explained about 86% of the variation in police shootings with multiple variables. The variable that influenced the model the most was the population of the county, with a correlation coefficient(r) of .94, followed by the number of hate crimes, the percent of the population being Asian, and the percent of the county being urban/rural, with r-values of .63, .34, and .32 respectively. However, outside of the model, when comparing variables to the number of shootings in proportion to the population, I found that economic conditions, such as poverty, unemployment rate, and per capita income had the largest r-values. From this I conclude that when trying to determine the number of shootings in a given county, factors such as population, hate crimes, and other demographics will give the most accurate prediction, but when trying to explain why some counties have unusually high rates of police violence in proportion to the population, economic conditions provide more insight.
